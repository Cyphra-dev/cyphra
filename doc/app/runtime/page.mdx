# Cyphra Client

**`CyphraClient`** (`@cyphra/runtime`) is a thin facade over the official **[neo4j-driver](https://www.npmjs.com/package/neo4j-driver)**. It connects Cyphra’s **compiled queries** (`CompiledCypher` from **`cypher`** or **`SelectQuery.toCypher()`**) to **`Session`** and **`ManagedTransaction`** APIs, with helpers to map results to **plain objects**.

Prefer **`import { CyphraClient, … } from "cyphra"`** when you use the meta-package; **`@cyphra/runtime`** is equivalent.

## Construction

Prefer an explicit **`CyphraNeo4j`** adapter (same options as below, plus an optional injected **`Driver`**):

```ts
import { CyphraClient, CyphraNeo4j } from "cyphra";

const client = new CyphraClient({
  adapter: new CyphraNeo4j({
    uri: process.env.NEO4J_URI!,
    user: process.env.NEO4J_USER!,
    password: process.env.NEO4J_PASSWORD!,
    database: "neo4j",
  }),
  debug: false, // logs Cypher text + parameter keys only (never values)
  onQuery: ({ cypher, paramKeys, durationMs }) => {
    // optional: metrics / tracing — values are never passed here
  },
});
```

Legacy shorthand (still supported):

```ts
const client = new CyphraClient({
  uri: process.env.NEO4J_URI!,
  user: process.env.NEO4J_USER!,
  password: process.env.NEO4J_PASSWORD!,
  database: "neo4j",
  debug: false,
  driverConfig: undefined,
});
```

| Option                  | Purpose                                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------------------------- |
| **`adapter`**           | **`CyphraNeo4j`** instance (recommended).                                                              |
| **`uri`**               | Bolt / Neo4j URI (`neo4j://`, `bolt://`, `neo4j+s://`, …) when not using **`adapter`**.                 |
| **`user` / `password`** | Passed to **`neo4j.auth.basic`**.                                                                       |
| **`database`**          | Neo4j 4+ database name for **`session({ database })`**.                                                 |
| **`debug`**             | When **`true`**, logs compiled Cypher and **parameter key names** only.                                 |
| **`onQuery`**           | Called after each **`runCompiled`** / **`runCompiledTx`** with **Cypher text**, **param keys**, and **duration** (ms). |
| **`driverConfig`**      | **`Config`** forwarded to **`neo4j.driver`** (pool size, `connectionAcquisitionTimeout`, `logging`, …). |

**`rawDriver`** exposes the underlying **`Driver`** for advanced scenarios.

Always call **`await client.close()`** on shutdown (graceful pool teardown).

## Sessions

- **`session()`** — you **must** **`close()`** the session when done.
- **`withSession(fn)`** — preferred; session is **always** closed after `fn` resolves or throws.

```ts
import { cypher } from "cyphra";

await client.withSession(async (session) => {
  await client.runCompiled(session, cypher`RETURN 1 AS n`);
});
```

## Transactions

- **`withReadTransaction(fn)`** — **`session.executeRead`**.
- **`withWriteTransaction(fn)`** — **`session.executeWrite`**.

Inside the callback, use **`*Tx`** methods so work runs on the **`ManagedTransaction`** (`tx.run`, retries, etc.):

```ts
import { cypher } from "cyphra";

await client.withWriteTransaction(async (tx) => {
  await client.runCompiledTx(tx, cypher`CREATE (n:Tmp { id: ${id} })`);
});
```

## Executing queries

| Session API                              | Transaction API                              | Description                                              |
| ---------------------------------------- | -------------------------------------------- | -------------------------------------------------------- |
| **`runCompiled`**                        | **`runCompiledTx`**                          | Run **`CompiledCypher`**.                                |
| **`runCypher`**                          | **`runCypherTx`**                            | Tagged template → compile → run.                         |
| **`queryRecords`** / **`queryRecord`**   | **`queryRecordsTx`** / **`queryRecordTx`**   | Same + **`toPlainRecords` / `toPlainRecord`**.           |
| **`selectRecords`** / **`selectRecord`** | **`selectRecordsTx`** / **`selectRecordTx`** | **`SelectQuery`** → **`toCypher()`** then same as above. |

Lower-level **`toPlainRecords`** / **`toPlainRecord`** are exported if you call **`session.run`** yourself.

## TypeScript and Neo4j types

Cyphra does **not** map Neo4j temporal/spatial types to custom classes in the query layer—what you get follows **`neo4j-driver`** and your **`toPlainRecord(s)`** usage. Shape results explicitly in application code where needed.

## Deployment

**`CyphraClient`** works in typical Node servers, workers, and many serverless environments where **long-lived drivers** are allowed. For **per-invocation** serverless, prefer **one driver per container reuse** pattern or connection pooling guidance from Neo4j—Cyphra does not add its own pool beyond the driver’s.
