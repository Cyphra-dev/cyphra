# Queries

Cyphra‚Äôs query layer is the **Cypher-JS / Drizzle-like** track: you compose **structures in TypeScript** (tagged templates, fluent builders, small compilers like **`compileRootOptionalOutgoingSelect`**) and always end up with **`CompiledCypher`** ‚Äî **text + parameters** for **`session.run`**. Nothing here **executes** the database; execution stays in **`CyphraClient`** + the official driver.

Interpolated values are **never** concatenated into the query string‚Äîthey are bound as **Neo4j parameters** (`$p0`, `$p1`, ‚Ä¶).

Imports:

```ts
import { cypher, select, node, rel, prop, eq, exists } from "cyphra";
import { query } from "cyphra"; // query.cypher, query.select, ‚Ä¶
// or: from "@cyphra/query" / "cyphra/query"
```

## Tagged Cypher

The **`cypher`** tag compiles to **`{ text, params }`** (`CompiledCypher`):

```ts
import { cypher } from "cyphra";

const compiled = cypher`
  MATCH (u:User) WHERE u.id = ${userId} RETURN u
`;
// compiled.text  ‚Üí MATCH (u:User) WHERE u.id = $p0 RETURN u
// compiled.params ‚Üí { p0: userId }
```

Use it with **`CyphraClient.runCompiled`**, **`queryRecords`**, or the driver‚Äôs **`session.run`** directly.

**`compileCypher`** is the underlying function if you need the same behavior without the tag.

## `CALL { }` subqueries

**`callSubqueryCompiled(inner)`** wraps **`inner.text`** as **`CALL { ‚Ä¶ }`** and rewrites **`inner`**‚Äôs parameters to **`$c0_p0`**, **`$c0_p1`**, ‚Ä¶ so they do not clash with an outer fragment‚Äôs **`$p0`**. Use **`branchIndex`** when you place several subqueries in one query (**`c1_`**, **`c2_`**, ‚Ä¶).

Lower-level **`remapCompiledCypherParamKeys(compiled, keyPrefix)`** (prefix must end with **`_`**) is the same rewriting primitive used by **`unionAllCompiled`**; use it if you assemble custom multi-part queries.

**`concatCompiledCypher([a, b, ‚Ä¶], { separator? })`** concatenates fragments: the first keeps **`p0`**, **`p1`**, ‚Ä¶; each next part is remapped with **`f1_`**, **`f2_`**, ‚Ä¶ so you can stitch clauses without manual param merging.

```ts
import { callSubqueryCompiled, cypher, select, concatCompiledCypher } from "cyphra";

const inner = select().match("(x:X)").returnStar().toCypher();
const sub = callSubqueryCompiled(inner);
const outer = cypher`MATCH (root:Root) WHERE root.id = ${"r1"}`;
const q = concatCompiledCypher([outer, sub]);
```

## SelectQuery builder

For structured reads without hand-writing every clause, use **`select()`** and pattern helpers:

```ts
import { node, rel, prop, eq, select } from "cyphra";

const u = node("User", "u");
const o = node("Organization", "o");
const m = rel("MEMBER_OF", "m");

const q = select()
  .match(u.out(m, o))
  .where(eq(prop(m.alias, "role"), "admin"))
  .returnFields({ id: prop(u.alias, "id") })
  .orderBy({ prop: prop(u.alias, "createdAt"), direction: "DESC" })
  .skip(0)
  .limit(20);

const compiled = q.toCypher();
```

Patterns:

- **`node("Label", "alias")`** ‚Äî node variable.
- **`rel("TYPE", "alias")`** ‚Äî single-hop relationship variable.
- **`rel("TYPE", "alias", varLength)`** ‚Äî variable-length segment: **`"any"`** (`*`), an integer (exact hops), or **`{ min?, max? }`** (at least one bound) for Neo4j **`[*]`**, **`*n`**, **`*min..max`**, **`*..max`**, **`*min..`** on the relationship.
- **`u.out(m, o)`**, **`u.in(m, o)`**, **`u.both(m, o)`** ‚Äî directed / undirected patterns.
- After **`match(...)`**, chain **`optionalMatch(pattern)`** to append **`OPTIONAL MATCH`** clauses in order.

### Predicates (`where`)

**`where`** accepts multiple predicates **AND**-combined: **`eq`**, **`neq`**, **`gt`**, **`gte`**, **`lt`**, **`lte`**, **`between`** (inclusive), **`isNull`**, **`isNotNull`**, **`inList`**, **`startsWith`**, **`endsWith`**, **`contains`**, **`matches`** (`=~`, pattern always parameterized), **`exists`** (Neo4j **`EXISTS { MATCH ‚Ä¶ }`** with optional inner **`WHERE`**), **`not`**, **`and`**, **`or`**.  
The left-hand side of comparisons is normally **`prop(alias, "field")`**; **`bareVar("x")`** emits the variable **`x`** alone (for list-comprehension filters and similar).  
Every value becomes a parameter except pure **`IS NULL` / `IS NOT NULL`**.

```ts
import { exists, prop, eq, select } from "cyphra";

select()
  .match("(u:User)")
  .where(exists("(u)-[:POSTED]->(p:Post)", eq(prop("p", "published"), true)))
  .returnStar();
```

### Optional `WITH` before `WHERE`

**`withOrderLimit(variables, order, limit)`** emits **`WITH ‚Ä¶ ORDER BY ‚Ä¶ LIMIT $pN`** after all **`MATCH` / `OPTIONAL MATCH`** clauses and **before** the outer **`WHERE`**. Use it for top‚ÄëN filtering per match (e.g. latest posts) while keeping the rest of the query parameter-safe.

### Return and modifiers

- **`returnFields({ ‚Ä¶ })`** or **`returnStar()`** is **required** before **`toCypher()`**.
- **`returnDistinct()`** after either emits **`RETURN DISTINCT ‚Ä¶`**.
- **`orderBy`**, **`skip`**, **`limit`** chain as in Cypher. Each **`orderBy`** entry may set **`nulls: "FIRST"`** or **`"LAST"`** (Neo4j **`NULLS FIRST` / `NULLS LAST`**). The same shape is used in **`withOrderLimit(..., order, limit)`**.

## Write helpers (`@cyphra/query`)

Small **parameterized** mutators complement **`cypher`** and migrations:

- **`compileCreate`**, **`compileMergeSet`**, **`compileSetWhere`**, **`compileDetachDeleteWhere`** ‚Äî node create / merge / patch / delete (detach).
- **Relationship edges (endpoints bound earlier):** **`compileMergeRelationship` / `compileCreateRelationship`** ‚Äî **`(from)-[rel:TYPE]->(to)`**; **`compileMergeRelationshipIn` / `compileCreateRelationshipIn(end, rel, type, start)`** ‚Äî **`(end)<-[rel:TYPE]-(start)`** (edge **start ‚Üí end**). Optional **`SET rel += $props`** when **`props`** is non-empty.
- **`compileDeleteWhere`** ‚Äî **`DELETE`** without **`DETACH`** (fails if relationships remain).
- **`compileDeleteRelationshipWhere(type, relAlias, predicates, { ends? })`** ‚Äî **`MATCH ()-[rel:TYPE]-() WHERE ‚Ä¶ DELETE rel`** with anonymous endpoints; **`ends`** is **`"out"`** (default), **`"in"`**, or **`"undirected"`**.
- **`compileSetRelationshipWhere`** ‚Äî same **`MATCH`** shape, **`SET rel += $props`** (patch by predicate).
- **`compileRemoveRelationshipProperties`** ‚Äî same **`MATCH`** shape, **`REMOVE rel.key`** for listed property names.
- **`compileRemoveProperties`** ‚Äî **`REMOVE n.key`** for listed scalar property names (nodes).
- **`compileUnwindMergeSet`** ‚Äî **`UNWIND $rows AS row MERGE ‚Ä¶ SET ‚Ä¶`** for batched upserts (`row.patch` map optional).
- **`compileForeachMergeSet`** ‚Äî same row shape as **`compileUnwindMergeSet`**, but **`FOREACH (row IN $rows | MERGE ‚Ä¶ SET ‚Ä¶)`** when you must not **`UNWIND`** the outer stream.
- **`compileForeachDeleteRelationshipsFromPath`** ‚Äî **`FOREACH (r IN relationships(p) | DELETE r)`** (no params); prefix with a **`MATCH`** that binds the path as **`p`** (for example after **`shortestPath`**).
- **`compileUnwindDetachDelete`** ‚Äî **`UNWIND $ids AS id MATCH ‚Ä¶ WHERE ‚Ä¶ DETACH DELETE`** for batched deletes by a key field.

Use **`runCompiledBatchWrite`** from **`@cyphra/orm`** to run several compiled steps in one write transaction.

## `LOAD CSV` fragment (C-Œ≥)

**`compileLoadCsvFrom(url, rowAlias, options?)`** emits **`LOAD CSV [WITH HEADERS] FROM $‚Ä¶ AS rowAlias`**, with **`url`** always bound as a parameter (default key **`csvUrl`**). Optional **`fieldTerminator`** is one of **`"comma"`**, **`"semicolon"`**, **`"pipe"`**, **`"tab"`** ‚Äî emitted as a safe literal, not arbitrary user text.

Append the rest of the import (e.g. **`MERGE`**, **`WITH row LIMIT ‚Ä¶`**) via **`cypher`** or string composition; check Neo4j‚Äôs rules for **`file:///`** vs remote URLs and Aura restrictions.

```ts
import { compileLoadCsvFrom, cypher } from "cyphra";

const load = compileLoadCsvFrom(csvUrl, "row");
const full = {
  text: `${load.text} MERGE (n:Person { id: row.id }) SET n.name = row.name`,
  params: load.params,
};
```

## Planner hints (C-Œ≥)

**`matchHintUsingIndex(variable, labelOrRelType, properties)`**, **`matchHintUsingScan(variable, labelOrRelType)`**, and **`matchHintUsingJoin([v1, v2, ‚Ä¶])`** return validated **`USING ‚Ä¶`** fragments (identifiers only ‚Äî no user-controlled Cypher injection). Place them after the **`MATCH`** pattern and before **`WHERE`**. Because **`concatCompiledCypher`** remaps parameters on every fragment after the first, either merge **`MATCH` + hint + tail** yourself when the tail binds values, or keep parameterized clauses in the first fragment only.

```ts
import { cypher, matchHintUsingIndex } from "cyphra";

const head = cypher`MATCH (n:Person)`;
const tail = cypher` WHERE n.email = ${addr} RETURN n`;
const hint = matchHintUsingIndex("n", "Person", ["email"]);
const q = {
  text: `${head.text} ${hint}${tail.text}`,
  params: { ...head.params, ...tail.params },
};
```

Hints are easy to misuse; validate plans with **`EXPLAIN`** / **`PROFILE`** on your target Neo4j version.

## List comprehension **`FILTER`** (C-Œ≥)

**`compileComprehensionFilterClause(itemVariable, listParamKey, listValue, predicates)`** builds **`item IN $listParamKey WHERE ‚Ä¶`** for use inside **`[ ‚Ä¶ | expr ]`**. Use **`bareVar("item")`** in predicates when the list element is a scalar; use **`prop("item", "field")`** when elements are maps or graph values with properties.

```ts
import { bareVar, compileComprehensionFilterClause, gt } from "cyphra";

const filter = compileComprehensionFilterClause("n", "nums", [1, 2, 3], [gt(bareVar("n"), 0)]);
const q = {
  text: `RETURN [ ${filter.text} | n * 2 ] AS doubled`,
  params: filter.params,
};
```

Cypher 25 **`FILTER`** / **`WHEN`** / **`NEXT`** outside this pattern are still written with **`cypher`** until version-specific emit exists.

## `CASE` expression

**`compileCaseExpression([{ when, then }, ‚Ä¶], elseThen)`** builds **`CASE WHEN ‚Ä¶ THEN ‚Ä¶ ELSE ‚Ä¶ END`**: each **`when`** array is AND-combined like **`where`**; comparison values in **`WHEN`** and all **`then`** / **`elseThen`** results are parameters. Wrap the result in **`RETURN ‚Ä¶`** or **`WITH ‚Ä¶`** yourself.

```ts
import { bareVar, compileCaseExpression, gt, lt } from "cyphra";

const kind = compileCaseExpression(
  [
    { when: [gt(bareVar("score"), 80)], then: "high" },
    { when: [lt(bareVar("score"), 40)], then: "low" },
  ],
  "mid",
);
const q = { text: `RETURN ${kind.text} AS band`, params: kind.params };
```

## Runtime shortcuts

**`CyphraClient`** exposes **`queryRecords`**, **`queryRecord`**, **`selectRecords`**, and **`selectRecord`** so you rarely wire **`run`** + **`toPlainRecords`** manually. Transaction variants **`*Tx`** exist for use inside **`withReadTransaction`** / **`withWriteTransaction`**. See [Cyphra Client](/runtime).

## Compared to Prisma or Drizzle

- **Prisma** generates a **client per schema**; Cyphra ships a **fixed** **`CyphraClient`** and optional **`createNodeCrud`** driven by a **parsed** **`.cyphra`** document‚Äîyour graph model stays a **text** artifact in the repo, closer to how [Drizzle](https://orm.drizzle.team/) keeps TypeScript schema files, but with **`cyphra.config.ts`** for CLI paths like Drizzle‚Äôs config file.
- **Relational** ORMs hide SQL; Cyphra keeps **Cypher visible** (`cypher`, builder output, **`EXPLAIN`**). Use **`cypher`** whenever the typed layer does not yet cover a clause‚Äîsee [Cypher coverage](/cypher-coverage).

## Result mapping

**`toPlainRecords`** / **`toPlainRecord`** map driver **`Result`** rows to plain objects (`record.toObject()`). There is no ORM identity map‚Äîjust data.
