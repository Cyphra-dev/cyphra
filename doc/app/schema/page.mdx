# Schema DSL

Define nodes, relationships, and decorators in `schema.cyphra`:

```cyphra
node User {
  id    String @id @default(cuid())
  email String @unique
  name  String @index
}
```

**Decorators (V1):** `@id`, `@unique`, `@index` (Neo4j 5+ range index on the node property), `@default(…)`, `@relationship(…)`. You cannot combine `@index` with `@id` or `@unique` on the same field. Relationship property indexes are not generated from the schema yet—use migrations.

The parser produces an AST used by `cyphra push`, `cyphra generate`, and programmatic APIs. Use **`printSchemaDocument`** from `@cyphra/schema` (or **`cyphra schema print`**) to emit a canonical `.cyphra` text for diffs and reviews—comments are not preserved.
