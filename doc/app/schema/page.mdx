# Schema DSL

Define nodes, relationships, and decorators in `schema.cyphra`:

```cyphra
node User {
  id    String @id @default(cuid())
  email String @unique
  name  String @index
}

relationship Follows {
  type "FOLLOWS"
  from User
  to User
  since DateTime @index
}
```

**Decorators (V1):** `@id`, `@unique`, `@index` (Neo4j 5+ **range** index: on nodes via `(n:Label)`, on rich relationships via `()-[r:TYPE]-()`), `@default(…)`, `@relationship(…)`. On **nodes**, you cannot combine `@index` with `@id` or `@unique` on the same field (those constraints already imply an index).

The parser produces an AST used by `cyphra push`, `cyphra generate`, and programmatic APIs. Use **`printSchemaDocument`** from `@cyphra/schema` (or **`cyphra schema print`**) to emit a canonical `.cyphra` text for diffs and reviews—comments are not preserved.
